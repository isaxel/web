## 1. Платформа Java EE (Jakarta EE)

**Java Enterprise Edition** — это набор спецификаций (стандартов) для разработки корпоративного ПО. Сейчас она развивается под названием **Jakarta EE**.

* **Спецификации:** Это просто документы (интерфейсы), описывающие, как должна работать та или иная часть системы (например, CDI, JPA, JAX-RS).
* **Реализации (Application Servers):** Конкретные программы, которые воплощают эти спецификации в коде.
* *Примеры:* **WildFly** (JBoss), **GlassFish**, **Payara**, **Open Liberty**.



### Принципы IoC, CDI и Location Transparency

* **IoC (Inversion of Control):** Вместо того чтобы объект сам создавал свои зависимости, он получает их извне.
* **CDI (Contexts and Dependency Injection):** Спецификация для управления жизненным циклом компонентов и внедрения зависимостей. Главная аннотация — `@Inject`.
* **Location Transparency:** Клиент не должен знать, где находится компонент (в той же JVM или на другом сервере). Это реализуется через JNDI или удаленные вызовы (RMI).

### Компоненты, Контейнеры и Жизненный цикл

* **Контейнер:** Среда (часть сервера приложений), которая управляет компонентами (EJB-контейнер, Web-контейнер). Он берет на себя транзакции, безопасность и **управление жизненным циклом** (создание, инициализация `@PostConstruct`, уничтожение `@PreDestroy`).
* **Дескрипторы развёртывания:** XML-файлы (`web.xml`, `ejb-jar.xml`), которые говорят контейнеру, как настраивать приложение. В современных версиях почти полностью заменены аннотациями.

### Профили платформы

1. **Full Profile:** Полный набор спецификаций (включая RMI, JMS).
2. **Web Profile:** Облегченная версия для веб-приложений (Servlets, JSF, JPA, CDI, EJB Lite).

## 2. Компоненты EJB (Enterprise JavaBeans)

EJB — это серверные компоненты для реализации бизнес-логики.

* **Stateless Session Beans:** Не хранят состояние клиента. Вызов метода — и "забыл". Максимально масштабируемы.
* **Stateful Session Beans:** Хранят состояние конкретного клиента между вызовами.
* **EJB Lite vs Full:** Lite — это подмножество функций для Web Profile (без удаленного доступа и очередей).

---

## 3. Взаимодействие и сообщения

* **JavaMail API:** Стандарт для отправки и получения почты (SMTP, IMAP, POP3).
* **JMS (Java Message Service):** Асинхронный обмен сообщениями.
* **Очереди (Queue):** Точка-точка (один отправитель — один получатель).
* **Топики (Topic):** Издатель-подписчик (один ко многим).


* **Message-Driven Beans (MDB):** Особый вид EJB, который "слушает" сообщения из JMS и реагирует на них.

---

## 4. Транзакции и Веб-сервисы

* **Транзакция:** Группа операций, которая выполняется либо полностью (commit), либо не выполняется вовсе (rollback).
* **JTA (Java Transaction API):** Позволяет управлять распределенными транзакциями (например, запись в БД и отправка в очередь одновременно).
* **Веб-сервисы:**
* **JAX-RS:** RESTful сервисы (JSON/XML, HTTP-методы GET, POST и т.д.).
* **JAX-WS:** SOAP-сервисы (тяжеловесные, основаны на XML и WSDL).

## 5. Платформа Spring

Spring — это де-факто стандарт в Java, альтернатива (и дополнение) к Jakarta EE.

* **Сходства:** Оба решают задачи DI, транзакций и веба.
* **Отличия:** Spring более модульный, "легковесный" (не требует тяжелого сервера приложений) и предлагает гораздо больше готовых интеграций.
* **Модули:** * **Core:** Основа (IoC/DI).
* **Spring Data:** Упрощенная работа с БД (репозитории).
* **Spring Security:** Мощная система аутентификации и авторизации.



### IoC и CDI в Spring

В Spring вместо `@Inject` чаще используется `@Autowired`. Spring Container (ApplicationContext) сканирует классы (`@Component`, `@Service`) и сам собирает "пазл" приложения.

## 6. Frontend: React JS

**React** — это библиотека (не фреймворк!) для создания пользовательских интерфейсов.

* **Архитектура:** Компонентный подход. Все приложение — это дерево компонентов. Используется **Virtual DOM** для быстрой отрисовки.
* **State vs Props:** * `props` — неизменяемые данные, приходящие снаружи (как аргументы функции).
* `state` — внутреннее состояние компонента, которое может меняться.


* **"Умные" (Smart) и "глупые" (Dumb) компоненты:**
* Dumb: Просто отображают данные (UI-компоненты).
* Smart: Содержат логику, делают запросы к API, управляют состоянием.


* **JSX:** Синтаксис, позволяющий писать HTML-подобный код прямо в JavaScript.
* **ReactRouter:** Библиотека для навигации между страницами без перезагрузки браузера (SPA).
* **Redux:** Глобальное хранилище данных (Store). Работает по схеме: *Action -> Reducer -> Store -> View*.

## 7. Frontend: Angular

**Angular** — это полноценный фреймворк от Google. Если React — это набор инструментов, то Angular — это завод.

* **Архитектура:** Основана на модулях (`NgModule`), компонентах и сервисах. Пишется на **TypeScript**.
* **DI (Dependency Injection):** В Angular мощная встроенная система внедрения зависимостей (сервисы внедряются в компоненты через конструктор).
* **Жизненный цикл:** `ngOnInit`, `ngOnChanges`, `ngOnDestroy` и др.
* **Шаблоны:** HTML с использованием директив (например, `*ngIf`, `*ngFor`).
* **Взаимодействие:** Используется класс `HttpClient` для запросов к серверу. Формы бывают двух видов: *Template-driven* (простые) и *Reactive* (сложные, с валидацией через код).
